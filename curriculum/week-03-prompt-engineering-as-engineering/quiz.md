**Q7.** What is the honest 2026 position on chain-of-thought (CoT)?

**Q8.** Turpin et al. (2023) showed that a model's visible chain-of-thought is not always faithful. What is the engineering consequence?

**Q11.** Prompt injection is hard to fully fix. What property of the model is the root cause?

**Q12.** Which is the more dangerous class of injection and why?

**Q13.** What is the division of labor between promptfoo and Langfuse prompt management?

**Q14.** What distinguishes a spec-then-implement loop (Claude Code / Cursor) from vibe-coding a prompt?

- A) Nothing; they're the same.
- B) Spec-then-implement writes the contract (task, format, acceptance criteria) first, then implements against it, reviews the change as a *diff* against a checklist, and gates on a pass rate — vibe-coding accepts whatever the agent generates with no spec, diff, or test.

1. **B** — The prompt is a versioned, diffable, testable artifact; "better" is a measured pass-rate claim, not a vibe. (Lecture 1 §1.)

3. **B** — Open-ended output means you assert a property (category, refusal, JSON-valid), not an exact string. (Lecture 1 §2.)
4. **B** — Instructions → system, untrusted data → user, model output → assistant (never authored by you). (Lecture 1 §3.)
5. **B** — Assistant-prefill 400s on 2026 frontier models; steer with the system prompt and output-format config. (Lecture 1 §3.)
6. **B** — Exemplars are versioned prompt; trim ones that don't move the pass rate — you're paying tokens for no measured gain. (Lecture 1 §4.)
7. **C** — CoT is task- and model-shaped, hurt by irrelevant context, and not universal; test it on your own examples. (Lecture 1 §5.)
8. **B** — The visible CoT is not a faithful audit; use it to maybe improve the answer, not to certify why. (Lecture 1 §5.)
9. **B** — Sample N paths, majority-vote; cost is ~N× and it needs discrete, comparable answers. (Lecture 1 §6.)
10. **B** — Persona is mostly tone/format theater, can induce overconfidence or widen the jailbreak surface; test its pass-rate effect. (Lecture 2 §1.)

12. **B** — Indirect injection rides in on content the user didn't write, so a tool-equipped agent acts on an attack the user never typed. (Lecture 2 §2.)
13. **B** — promptfoo is the build-time gate; Langfuse is the runtime registry with labelled versions and instant rollback. (Lecture 2 §3–4.)
14. **B** — Spec first, implement, diff, review against a checklist, gate on a pass rate — vibe-coding skips all of that. (Lecture 2 §6.)
