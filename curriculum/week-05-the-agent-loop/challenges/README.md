# Week 5 — Challenges

The exercises drill the mechanics. **The challenge makes you the engineer who has to defend a build-vs-buy decision with numbers.** You re-implement your hand-rolled agent on top of the `claude-agent-sdk`, run both against the same benchmark, and report pass rate, cost, code surface, and what the SDK hides — honestly, with the wrinkles.

## Index

1. **[Challenge 1 — Hand-rolled vs the SDK](challenge-01-hand-rolled-vs-sdk.md)** — run the same agent two ways (your loop and the SDK runner) on a shared task set, measure four numbers, and write the recommendation a senior engineer would actually defend. (~90 min)

Challenges are optional for passing the week, but this one is the single best preparation for Week 13, where you re-implement this agent *again* as a LangGraph graph and have to say what you gained. The skill — comparing two ways of doing the same thing with measured numbers instead of taste — is exactly what separates the engineer who picks a framework because it's popular from the one who picks it because the numbers said so.
